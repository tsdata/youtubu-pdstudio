## 시리즈 1 · Advanced RAG 전략

단순 벡터 검색을 넘어 **프로덕션 수준의 RAG 파이프라인**을 설계하는 실무 테크닉.

### EP 01 — Hybrid Search 구현하기

- **썸네일 훅:** "Keyword vs Vector? 왜 하나만 쓰면 망하는가"
- 키워드 검색(BM25)과 벡터 검색의 장단점 비교
- Reciprocal Rank Fusion(RRF)으로 두 결과를 합치는 원리
- Elasticsearch / Weaviate / Pinecone 환경별 구현 데모
- 실험: 하이브리드 적용 전후 Recall@10 비교

### EP 02 — Re-ranking 적용기

- **썸네일 훅:** "검색 결과 100개 중 진짜 필요한 5개만 고르기"
- Cross-Encoder 기반 Re-ranker 원리 (Cohere Rerank, BGE Reranker)
- Two-Stage Retrieval 아키텍처 설계
- 지연 시간(Latency) vs 정확도 트레이드오프 분석
- 코드 워크스루: LangChain + Cohere Rerank 통합

### EP 03 — Graph RAG

- **썸네일 훅:** "단순 RAG와 Graph RAG의 결정적 차이"
- 엔티티-관계 그래프 자동 추출 파이프라인
- Neo4j / NetworkX를 활용한 지식 그래프 구축
- 그래프 순회 + 벡터 검색 결합 전략
- 유스케이스: 법률 문서, 조직도, 기술 의존성 분석

## 시리즈 2 · 에이전트 하네스 & 평가 체계

에이전트를 **"만드는 법"이 아니라 "잘 만드는 법"**에 집중하는 시리즈.

### EP 04 — Agent Harness를 활용한 성능 평가

- **썸네일 훅:** "내 에이전트는 얼마나 똑똑할까?"
- Agent Harness 개념 정의와 필요성
- 테스트 시나리오 설계: 단일 턴 vs 멀티 턴 평가
- 성공률, 정확도, 비용, 지연 시간 — 핵심 메트릭 4가지
- 자동화된 회귀 테스트 파이프라인 구축

### EP 05 — GAIA·SWE-bench를 내 프로젝트에 적용하는 법

- **썸네일 훅:** "에이전트 성능 측정의 표준을 내 프로젝트에"
- GAIA 벤치마크 구조 분석과 커스텀 데이터셋 제작
- SWE-bench로 코딩 에이전트 평가하기
- 사내 벤치마크 프레임워크 설계 가이드
- 결과 시각화: 리더보드 대시보드 만들기

### EP 06 — 에이전트 디버깅 & 모니터링

- **썸네일 훅:** "에이전트가 루프에 빠졌을 때 탈출하는 법"
- Langfuse / LangSmith로 추론 과정 트레이싱
- 무한 루프·환각·도구 오용 패턴 탐지
- 가드레일 설계: 최대 반복 횟수, 타임아웃, Fallback 전략
- 실시간 모니터링 대시보드 구축 데모

## 시리즈 3 · 컨텍스트 엔지니어링

프롬프트 엔지니어링의 상위 개념 — 모델의 **"기억력"과 "이해력"**을 최적화하는 기술.

### EP 07 — Context Compression

- **썸네일 훅:** "긴 문서를 토큰 낭비 없이 쑤셔 넣는 압축 기술"
- 요약 기반 압축 vs 추출 기반 압축
- LLMLingua, Selective Context 등 압축 라이브러리 비교
- 압축률 vs 정보 손실 실험 결과
- 실전 적용: 10만 토큰 문서를 2만 토큰으로 압축하기

### EP 08 — JSON Mode & Function Calling

- **썸네일 훅:** "LLM에게 깔끔한 결과값을 받아내는 컨텍스트 설계법"
- 구조화된 출력을 유도하는 프롬프트 패턴
- Pydantic + Instructor로 타입 안전한 LLM 호출
- Function Calling 스키마 설계 베스트 프랙티스
- 에러 핸들링: 파싱 실패 시 자동 재시도 전략

### EP 09 — Long Context vs RAG

- **썸네일 훅:** "200K 컨텍스트 시대, RAG가 정말 필요 없을까?"
- Needle In A Haystack 테스트 직접 수행
- Long Context의 실제 성능 한계 실험
- RAG vs Stuff-It-All-In: 비용·정확도·속도 3축 비교
- 결론: 언제 RAG를, 언제 Long Context를 선택할 것인가

## 시리즈 4 · 실전 에이전트 아키텍처

앞선 기술들을 **모두 조합**하는 End-to-End 프로젝트.

### EP 10 — 멀티 에이전트 시스템 심화

- **썸네일 훅:** "기획자 AI, 개발자 AI, 검수자 AI를 한 팀으로"
- 에이전트 역할 분리와 오케스트레이션 패턴
- CrewAI / AutoGen / LangGraph 프레임워크 비교
- 에이전트 간 통신 프로토콜 설계
- 실습: 3-에이전트 코드 리뷰 파이프라인

### EP 11 — 종합 프로젝트: 사규 마스터 에이전트

- **썸네일 훅:** "RAG + Context Engineering으로 우리 회사 사규를 완전 정복"
- 요구사항 분석 → 아키텍처 설계 → 구현 → 평가 전 과정
- Hybrid Search + Re-ranking + Graph RAG 통합
- 컨텍스트 압축으로 비용 최적화
- Agent Harness로 품질 검증 및 배포
