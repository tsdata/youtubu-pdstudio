## 시리즈 5 · MCP(Model Context Protocol) 실전

> 2026년 가장 뜨거운 키워드. MCP 서버를 직접 만들고 운영하는 채널은 아직 거의 없음.

### EP 12 — MCP 서버 직접 만들기: AI의 USB-C를 이해하다

- **썸네일 훅:** "API 연동 10개를 MCP 서버 1개로 대체한 방법"
- MCP란 무엇인가 — USB-C 비유로 3분 만에 이해하기
- FastMCP(Python)로 나만의 MCP 서버 구축 (사내 DB 연결)
- Claude Desktop / Claude Code에 MCP 서버 연결 데모
- 보안 고려사항: 인증, 권한 관리, Tool Poisoning 방지

### EP 13 — MCP 레지스트리와 거버넌스: Pinterest는 어떻게 했나

- **썸네일 훅:** "MCP 서버 50개를 관리하는 Pinterest의 비밀"
- 도메인별 MCP 서버 분리 전략 (Presto, Spark, Airflow 각각)
- 중앙 레지스트리 설계: 승인된 서버 관리 + API 디스커버리
- Human-in-the-Loop 승인 워크플로우 구현
- 실습: 사내 MCP 서버 카탈로그 만들기

### EP 14 — Agent-to-Agent 통신: MCP + A2A 프로토콜

- **썸네일 훅:** "에이전트끼리 대화하는 시대가 왔다"
- MCP(도구 연결) vs A2A(에이전트 간 통신) 역할 구분
- Google A2A 프로토콜 구조 분석
- 멀티 벤더 에이전트 협업 시나리오 설계
- 실습: MCP + A2A 결합 멀티 에이전트 파이프라인

## 시리즈 6 · LLM FinOps — 에이전트 비용 최적화

> 2026년 기업들의 가장 큰 고민. "AI가 돈을 벌어주는 게 아니라 태우고 있다"
> 이 주제를 다루는 한국어 채널은 사실상 전무

### EP 15 — 에이전트 토큰 경제학: 왜 비용이 10배로 폭발하는가

- **썸네일 훅:** "에이전트 1회 호출에 $8? 비용 폭탄의 진짜 원인"
- 에이전트 vs 챗봇 — 왜 3~10배 더 비싼가 (Multi-turn Loop 분석)
- 입력/출력 토큰 비대칭 (출력이 4~8배 비싼 이유)
- Context Accumulation 비용 시뮬레이션
- 비용 추적 대시보드 구축 (Langfuse 비용 태깅)

### EP 16 — Model Router: 요청마다 최적 모델을 자동 선택하기

- **썸네일 훅:** "단순 질문에 GPT-4o를 쓰면 190배 손해"
- 계층형 모델 전략: Frontier → Mid-tier → Small 모델
- Plan-and-Execute 패턴으로 비용 90% 절감 사례
- LiteLLM / Portkey 기반 모델 라우터 구현
- 실습: 질문 복잡도 분류기 + 자동 라우팅 시스템

### EP 17 — Semantic Caching과 Prompt Compression

- **썸네일 훅:** "같은 질문에 왜 매번 돈을 내는가: 캐싱으로 95% 절약"
- Semantic Cache: 유사 질문을 캐시에서 즉시 반환
- Prompt Caching (KV 캐시): Claude/GPT의 네이티브 캐싱 활용
- EP07 Context Compression과의 결합 전략
- 실습: Redis + 임베딩 기반 Semantic Cache 구축

## 시리즈 7 · 프로덕션 에이전트 안전장치

> "데모에서는 잘 되는데 프로덕션에서 폭주합니다" — 실무자 최대 페인포인트

### EP 18 — Toxic Flow Analysis: 에이전트 데이터 흐름의 취약점 찾기

- **썸네일 훅:** "당신의 에이전트, 해커의 놀이터일 수 있습니다"
- Toxic Flow Analysis 프레임워크 (ThoughtWorks Technology Radar)
- Tool Poisoning / Cross-Server Shadowing 공격 시나리오
- Prompt Injection 방어 패턴 3가지
- 실습: 에이전트 보안 감사 체크리스트 + 자동 스캔 도구

### EP 19 — 에이전트 가드레일 고급편: 예산·시간·품질 삼중 제어

- **썸네일 훅:** "새벽 2시에 에이전트가 $3,000을 태웠습니다"
- 토큰 예산 제한 (Hard Limit + Graceful Degradation)
- 재귀 루프 자동 종료 (Iteration Cap + Timeout)
- 품질 게이트: 응답이 기준 미달이면 자동 재시도 vs 에스컬레이션
- 실습: LangGraph interrupt + 예산 미들웨어 구현

### EP 20 — 에이전트 메모리 아키텍처: 세션을 넘어서는 기억

- **썸네일 훅:** "어제 대화를 기억 못 하는 에이전트는 쓸모없다"
- Short-term(대화 내) vs Long-term(세션 간) 메모리 설계
- 메모리 저장 전략: Vector DB / KV Store / Markdown 파일
- 메모리 압축: 오래된 기억을 요약하여 토큰 절약
- 실습: Persistent Memory Agent 구현 (세션 간 컨텍스트 유지)

## 시리즈 8 · 실전 프로젝트 — 돈 되는 에이전트

> 기술을 종합하여 **실제 비즈니스 가치**를 만드는 프로젝트

### EP 21 — 코드 리뷰 에이전트: PR을 자동으로 검토하는 AI 팀원

- **썸네일 훅:** "이 에이전트가 시니어 개발자 리뷰 시간을 70% 줄였습니다"
- GitHub MCP 서버 + 코드 분석 에이전트 설계
- PR diff 파싱 → 보안 취약점 / 코드 스멜 / 컨벤션 체크
- SWE-bench 기반 성능 평가 (EP05 활용)
- 비용 최적화: 파일별 모델 라우팅 (EP16 활용)

### EP 22 — 회의록 → 액션 아이템 에이전트: 회의가 끝나면 일이 시작된다

- **썸네일 훅:** "30분 회의에서 자동으로 Jira 티켓 5개가 생성됩니다"
- 음성 → 텍스트 → 요약 → 액션 아이템 추출 파이프라인
- MCP로 Jira/Notion/Slack 자동 연동
- 멀티 에이전트: 요약 Agent + 분류 Agent + 등록 Agent (EP10 활용)
- 실제 회의록 3건으로 품질 평가 (EP04 Harness 활용)

### EP 23 — RAG + Agent로 만드는 기술 문서 챗봇 (프로덕션 배포까지)

- **썸네일 훅:** "사내 위키 2,000페이지를 AI가 1초 만에 답합니다"
- FastAPI + LangGraph 기반 프로덕션 아키텍처
- EP01~03 통합: Hybrid Search + Re-rank + Graph RAG
- EP07 + EP17 통합: Compression + Caching 비용 최적화
- Docker 배포 + Langfuse 모니터링 + CI/CD 파이프라인
